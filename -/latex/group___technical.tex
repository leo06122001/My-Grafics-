{\footnotesize\ttfamily \#define \+\_\+\+T\+X\+\_\+\+N\+O\+I\+N\+IT}



Запрет ранней инициализации T\+X\+Lib. 

\begin{DoxyVerb}     Если константа определена с помощью \#define @a до включения TXLib.h в программу, ранняя инициализация
     (до запуска функции @c main) @b не проводится.
\end{DoxyVerb}


\begin{DoxyNote}{Note}
Ранняя инициализация включает\+:
\begin{DoxyItemize}
\item Открытие окна консоли,
\item Установку кодовой страницы \+\_\+\+T\+X\+\_\+\+C\+O\+D\+E\+P\+A\+GE (1251) консоли для поддержки русского языка,
\item Установку русской локали стандартной библиотеки C++ (\char`\"{}\+Russian\char`\"{} / \char`\"{}ru\+\_\+\+R\+U.\+C\+P1251\char`\"{} и L\char`\"{}\+Russian\+\_\+\+Russia.\+A\+C\+P\char`\"{}),
\item Переинициализация библиотек stdio и iostream на случай, если приложение не скомпоновано как консольное и библиотеки остались неинициализированы,
\item Перехват системных сигналов (деление на 0, арифметические ошибки, обращение по неверному адресу и т.\+д.),
\item Перехват необработанных исключений,
\item Смена заголовка консольного окна,
\item Установка режима паузы по завершении программы, чтобы окно закрывалось не сразу,
\item Подавление паузы при запуске из сред программирования, заставляющей нажимать на клавишу два раза. ~\newline
 
\end{DoxyItemize}

Если ранняя инициализация запрещена, но в E\+X\+E-\/файле создается окно T\+X\+Lib с помощью \mbox{\hyperlink{group___drawing_gafce21ce98474d5559f316b3d47cdbfab}{tx\+Create\+Window()}}, то библиотека все равно будет инициализирована. В {\bfseries{D\+LL}} ранняя инициализация никогда {\bfseries{не}} проводится. @nn

Ранняя инициализация не потокобезопасна (not thread-\/safe).
\end{DoxyNote}

Включает/отключает внутреннюю трассировку исполнения кода библиотеки. 

\begin{DoxyVerb}     Трассировка идет через макрос TX_TRACE, который подставляется перед каждым оператором (statement).
     По умолчанию трассировка выключена.

     По умолчанию трассировка ведется через функцию OutputDebugString(), ее вывод можно перехватить
     утилитами-логгерами, например, <a href=http://technet.microsoft.com/ru-ru/sysinternals/bb896647%28en-us%29.aspx>
     DbgView</a>. Это можно изменить, переопределив макрос TX_TRACE до включения TXLib.h в программу.
\end{DoxyVerb}


\begin{DoxyWarning}{Warning}
Трассировка {\bfseries{очень}} тормозит выполнение программы, особенно при отладке в Microsoft Visual Studio. В этом случае лучше пользоваться Dbg\+View (см. выше) и запускать отлаживаемую программу отдельно, а не из-\/под отладчика Visual Studio.
\end{DoxyWarning}
\begin{DoxyNote}{Note}
\+\_\+\+T\+X\+\_\+\+A\+L\+L\+O\+W\+\_\+\+T\+R\+A\+CE и T\+X\+\_\+\+T\+R\+A\+CE задаются {\bfseries{перед}} включением \mbox{\hyperlink{_t_x_lib_8h}{T\+X\+Lib.\+h}} в программу.
\end{DoxyNote}

     утилитами-логгерами, например, <a href=http://technet.microsoft.com/ru-ru/sysinternals/bb896647%28en-us%29.aspx>
     DbgView</a>. Для "раскраски" лога есть файл <tt>TX\Dev\DbgView.ini,</tt> его надо загрузить в DbgView
     через меню Edit/Filter/Load (Ctrl+L).

     С помощью TX_TRACE можно трассировать код самой библиотеки TXLib. Для этого надо разрешить трассировку
     TXLib, определив макрос _TX_ALLOW_TRACE перед включением файла TXLib.h в программу. По умолчанию
     трассировка TXLib выключена.

     TX_TRACE можно переопределить в свой код. Тогда, если трассировка библиотеки разрешена, он будет
     вызываться почти перед каждой исполняемой строкой TXLib. Может быть, это кому-нибудь будет интересно.
\end{DoxyVerb}
